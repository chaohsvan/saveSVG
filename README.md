# saveSVG

这是一个非常高效的**“内容 -> 逻辑可视化 -> 自动化存储”**的工作流。你通过 Gemini 完成复杂的逻辑建模，再通过 Python 脚本解决了浏览器/编辑器手动保存 SVG 文件的繁琐步骤。

以下是基于你提供的提示词和脚本，为你梳理的完整操作流程说明：

---

## 🛠️ 全流程操作指南：从文字到矢量图

这个流程分为三个阶段：**AI 建模**、**本地脚本准备**、**快速同步保存**。

### 第一步：在 Gemini 中配置 “逻辑绘图专家” (Gem)

你提供的提示词是一个高质量的指令，它要求 Gemini 扮演一个学术风格的制图师。

1. **创建 Gem**：在 Gemini 界面创建一个新的 Gem，将你提供的**提示词（第1部分）**粘贴进去。
2. **输入内容**：当你需要分析复杂的文章、方案或代码逻辑时，直接将文字发给这个 Gem。
3. **获取代码**：Gemini 会自动分析逻辑并直接输出一段 `<svg>...</svg>` 格式的代码。
> **注意：** 由于提示词要求“只返回代码”，你会得到一个非常干净的输出，方便后续复制。



---

### 第二步：运行本地 Python 脚本 (准备接收)

你的 Python 脚本是一个**实时监听的“粘贴板过滤器”**。

1. **启动脚本**：在终端（Terminal 或 CMD）运行你的代码：
```bash
python your_script_name.py

```


2. **设置文件名**：
* 脚本会提示：`1. 请输入文件名`。
* 输入如 `research_flow`（不写后缀也可以，脚本会自动补齐 `.svg`）。


3. **等待输入**：看到 `2. 请粘贴 SVG 代码...` 的提示后，脚本进入等待状态。

---

### 第三步：复制、粘贴与触发保存

这是最关键的联动步骤，利用了脚本中的**单引号触发机制**。

1. **从 Gemini 复制**：点击 Gemini 输出代码块右上角的“复制”按钮。
2. **在终端粘贴**：回到运行脚本的终端，直接按下粘贴快捷键（`Ctrl+V` 或 `Cmd+V`）。
3. **添加触发符并回车**：
* **关键动作**：在粘贴完的长串代码最后，手动输入一个**单引号 `'**`（英文或中文单引号均可）。
* **敲击回车**：脚本识别到行尾的 `'`，会立即停止接收并封包。


4. **完成**：脚本会提示 `✅ 识别到结束符，保存成功！` 并显示文件的绝对路径。

## 💡 流程优点分析

| 维度 | 传统方式 | 你的方式 |
| --- | --- | --- |
| **排版审美** | 手动拖拽节点，耗时耗力 | Gemini 自动计算坐标，**学术风格**高度统一 |
| **文件处理** | 需新建文件、改后缀、粘贴、保存 | 脚本**自动创建**，文件名输入简单 |
| **大文件友好** | 终端粘贴大数据易卡死或截断 | 采用 `while True` 逐行读取，**稳定性更高** |

## 🌟 小贴士（进阶建议）

* **如何查看结果？** 生成的 `.svg` 文件可以直接拖进 Chrome、Edge 浏览器查看，或者用 VS Code 的插件实时预览。由于是矢量图，你可以无限放大而不模糊。
* **关于触发符的微调**：
如果你的 SVG 属性中本身含有单引号（虽少见，通常是双引号），请确保它是加在**最后一行回车前**。
* **Gemini 绘图限制**：
如果逻辑极其复杂（节点超过 20 个），SVG 的坐标计算可能会让代码变得非常长。如果 Gemini 输中途断了，记得对它说“继续输出”。
